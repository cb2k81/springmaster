# Changelog 000043: Springmaster Reference Project and Standards Strategy

## Summary

Documents the corrected Springmaster adoption strategy and introduces the master-level standards workstream.

## Changes

- Defines that existing projects are not supplied first by Springmaster.
- Defines Catalog-demo as the clean-room reference/demo app for Springmaster patterns.
- Defines CatalogItem as the first reference domain object.
- Defines Springmaster as owner of reusable standards, conventions, ADRs, reference implementations and validation gates.
- Adds the standards documentation root and API standards entry point.
- Records that IDM ADRs are source material for generic Springmaster standards, not direct target-project instructions.
- Keeps IDM, Personnel and other existing projects out of the immediate delivery path.

## Validation

Documentation-only patch. Maven tests are not required.
