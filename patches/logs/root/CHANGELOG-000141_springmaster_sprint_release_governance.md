# 000141_springmaster_sprint_release_governance

Closes the P0 sprint and release governance gap.

## Changes

- separates sprint, commit, patch artifact and product-version identities;
- replaces the historical version-policy journal with a compact current policy;
- defines sprint summaries, release qualification, hotfix and managed-target rules;
- adds a fail-closed release-manifest generator and non-pushing qualification command;
- adds deterministic release-manifest integration coverage;
- aligns the aggregate and Maven versions for the new release-tooling capability.

## Versions

- Platform: `0.15.0-foundation`
- Tooling: `0.5.0`
